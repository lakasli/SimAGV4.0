# 目标
把一个小功能实现为“原子”：边界清晰、独立可测、不会产生隐形依赖链。

# 约束
- 不调用任何其他原子/同层模块
- 输入输出契约明确（尽量无副作用）
- 复杂逻辑拆小：保持单模块可读
- 错误返回清晰（错误码/异常/Result类型按项目惯例）

# 工作流
1. 明确原子职责：用一句话描述“做什么”（避免复合职责）
2. 定义契约：输入、输出、错误、边界条件
3. 实现：保持短小、可读、可测
4. 给出单元测试用例表（用例名 + 输入 + 期望）

# 输出格式
- 原子模块代码
- 契约说明（I/O与错误）
- 单测用例清单（不必写全测试框架代码，除非用户要求）

# 质量检查清单
- 该模块能否脱离项目其余部分被理解？
- 是否引入了同层依赖？
- 是否存在“隐式流程控制”（把流程放进原子里）？
