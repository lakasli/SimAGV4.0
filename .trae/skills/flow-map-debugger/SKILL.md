# 目标
把排障从“读大量业务代码+追调用栈”变为“查流程地图+定位单点模块”。

# 工作流
1. 收集：入口输入、错误码/异常、执行到的stepId（或最后成功step）
2. 反查流程表：确定“应该执行到哪里”和“实际停在哪里”
3. 锁定模块：对应atom/molecule的契约与边界条件
4. 构造最小复现：只跑该step所需输入
5. 修复与验证：先单测原子，再集成测分子，再跑流程回归

# 输出
- stepId定位结论
- 怀疑点Top N（按概率排序：输入边界/数据转换/错误分支/外部依赖）
- 最小复现方案
- 验证清单（防回归）
