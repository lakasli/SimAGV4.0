# 目标
把“网状依赖”重构为“伞状/树状依赖”，消除隐形依赖链。

# 审计维度
- 同层互调（L4<->L4，L3<->L3）
- 下层调用上层（L4/L3调用L2/L1）
- 循环依赖
- 控制流下沉（L3/L4里出现流程编排、重试、路由、状态机）

# 输出
1) 违例清单：位置、类型、影响
2) 重构方案：拆分为哪些原子/分子、流程上提到L2的哪些step
3) 迁移顺序：先解环、再隔离同层、最后上提控制流
4) 回归测试点：哪些行为必须保持不变

# 重构策略（优先级）
1. 先切断循环依赖（引入L2端口/命令对象做反转）
2. 再消除同层互调（抽出公共逻辑为新的原子）
3. 最后上提控制流（把分支/重试/路由移到指挥官流程表）
